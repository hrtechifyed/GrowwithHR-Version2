## Applicability and evidence separation

M2 keeps applicability status separate from evidence status.

Supported applicability statuses include:

- `applicable`
- `likely-applicable`
- `not-currently-applicable`
- `more-information-needed`
- `specialist-review`

Supported evidence statuses include:

- `not-requested`
- `not-provided`
- `provided`
- `not-verified`
- `verified`

The current governed catalog defaults evidence to:

`not-requested`

The status `verified` must not be assigned without an approved verification process.
