## Purpose

M3 converts the governed M2 traceability bundle into one executive-facing Compliance Story without recreating recommendation logic in the presentation layer.

The private-beta story includes:

- a company snapshot derived from confirmed and derived facts;
- a Compliance DNA summary with separate safe-status counts;
- a maximum of three deterministically ranked priorities;
- every governed rule evaluation, including non-action and missing-information outcomes;
- trigger facts and missing information;
- rationale, next action, suggested timeline and implications;
- evidence status kept separate from applicability;
- structured source metadata and links;
- assumptions and limitations.

## Protected product boundary

M3 does not alter the public production path.

The stable route remains:

`/analyze-company.html`

The M3 experience remains private at:

`/analyze-company-v3.html`

The default feature flag remains:

`complianceDnaV3: false`

M3 must not:

- write a new browser-storage key;
- rename or delete a protected key;
- modify `growwithhr-advisory-briefing-v2`;
- modify the stable report record;
- modify PDF fields or generation;
- modify email or delivery payloads;
- change deterministic M2 rule decisions;
- claim verified compliance, verified coverage or legal certification;
- enable the private-beta route publicly.

## Safe status taxonomy

M3 renders the existing governed applicability statuses without reinterpretation:

- `applicable`
- `likely-applicable`
- `not-currently-applicable`
- `more-information-needed`
- `specialist-review`

Evidence remains a separate status dimension:

- `not-requested`
- `not-provided`
- `provided`
- `not-verified`
- `verified`

The current assessment does not verify documentary evidence. A `verified` state may only originate from a separately approved verification process.

## Implementation inventory

### M3 model schema

Path:

`data/schema/compliance-story.schema.v1.json`

The schema freezes the private-beta model version `1.0.0`. It governs the company snapshot, Compliance DNA counts, top-priority references, obligations, groups, assumptions, sources, limitations and protected-contract metadata.

### Pure Compliance Story model

Path:

`js/assessment-v3/compliance-story-model.js`

The model:

- accepts a complete M2 recommendation-traceability bundle;
- does not access the DOM, browser storage, network, PDF, email or delivery code;
- does not evaluate rules;
- creates one obligation for every M2 rule evaluation;
- retains recommendation identifiers when available;
- preserves safe applicability and evidence statuses;
- ranks no more than three priorities deterministically;
- freezes the returned model;
- sets `privateBetaOnly: true` and `stableReportMutation: false`.

### Private-beta presentation controller

Path:

`js/assessment-v3/compliance-story-presentation.js`

The controller:

- consumes only the in-memory result exposed by the M2 diagnostics controller;
- injects the isolated M3 private-beta section on the v3 route;
- renders the snapshot, status counts, top priorities, grouped findings, assumptions, sources and limitations;
- supports loading, ready, empty and safe error states;
- listens for M2 refresh events;
- does not read or write browser storage;
- exposes read-only controller state for maintained browser tests.

### Scoped presentation styles

Path:

`css/20-compliance-story.css`

The stylesheet is scoped to `.dna-compliance-story` and includes responsive layouts, keyboard focus treatment and reduced-motion behavior.

### Isolated route loader

Path:

`js/build-marker.js`

The existing build marker conditionally imports the M3 presentation only when `#dnaTraceability` is present. Other routes do not load the M3 module.

## Rollback

M3 is presentation-only and introduces no persistence migration.

Rollback steps:

1. Remove the conditional M3 import from `js/build-marker.js` or revert the M3 commits.
2. Keep `complianceDnaV3` disabled.
3. Continue routing public users to `/analyze-company.html`.
4. Preserve the M2 traceability files and protected v2 state unchanged.
5. No storage cleanup or data migration is required.

## Known limitations

- The initial governed recommendation catalog contains a limited set of deterministic advisory rules.
- Source links support further review and do not prove applicability.
- Assessment facts are user-supplied and are not independently verified.
- M3 does not yet replace the stable HTML report or stable PDF.
- Stable report/PDF integration requires a separately approved versioned migration.
- The private-beta route remains no-index and disabled by default for public routing.
