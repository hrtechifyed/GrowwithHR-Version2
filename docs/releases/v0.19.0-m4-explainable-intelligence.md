# GrowWithHR v0.19.0 — M4 Explainable Intelligence

Status: **release candidate implementation**  
Route: `/m4-explainable-intelligence.html`  
Public production route: unchanged  
Cloud persistence: not introduced

## Purpose

M4 converts the validated M2 traceability and M3 Compliance Story contracts into separate, explainable intelligence dimensions. It deliberately avoids a single blended percentage.

The implementation provides:

1. **Profile completeness** — the weighted completion of governed organisation facts.
2. **Applicability certainty** — the share of deterministic rule outcomes with supported statuses and no missing decision facts.
3. **Evidence coverage** — the share of governed findings or facts with an explicit available or verified evidence state.
4. **Information-gain questions** — unanswered governed questions ranked by the improvement they are expected to make to one named dimension.
5. **Reproducible decision trace** — a stable record of each calculation and why it was performed.

## Safe-intelligence rules

- No blended health, maturity or compliance score is produced.
- Applicability, completeness and evidence remain separate.
- The model is deterministic for the same input; only `generatedAt` changes.
- The private view reads protected browser records without writing to them.
- No cloud persistence, authentication or evidence upload is added.
- The result is not legal certification.
- Compliance interpretation remains restricted to India operations.

## Governed question catalogue

The first M4 catalogue covers legal entity, employee count, Indian operating states or union territories, work model, People/HR ownership and evidence-register availability. Each question names the dimension it improves and carries a documented information-gain weight.

## Acceptance evidence

The release candidate must demonstrate:

- three separate metrics, each constrained to 0–100;
- no `overallScore`, `overall` metric or blended percentage;
- deterministic output for the same protected input;
- targeted questions ranked by estimated information gain;
- a minimum five-step decision trace;
- a schema contract using JSON Schema 2020-12;
- no writes to protected storage;
- no change to the stable assessment, PDF, email or delivery routes.

## Release decision

This implementation is the M4 release candidate foundation. The canonical application version should remain unchanged until the complete repository regression suite, private-route browser validation and release approval are recorded. A coordinated `0.19.0` package/configuration cut should be performed only after those gates pass.

## Rollback

Remove or disable the private M4 route and model files. M3 remains complete, the public v2 route remains available, and no storage migration or cleanup is required.
